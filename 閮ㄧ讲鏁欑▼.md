# 世界树 · 部署教程

全程不需要写代码，跟着点就行。大约 20-30 分钟。

一共要用到三个免费网站：
- **GitHub** —— 存放代码
- **Supabase** —— 数据库（存建言、点赞、档案）
- **Vercel** —— 把网站发布上线

---

## 第一步：把代码放到 GitHub

1. 打开 [github.com](https://github.com)，注册/登录账号
2. 右上角 `+` → **New repository**
3. 名字随便填，比如 `worldtree`，选 **Private**（私有）或 Public 都行 → **Create repository**
4. 页面上会提示上传文件，点 **uploading an existing file**
5. 把这个文件夹里除了 `.env.local.example` 之外的所有文件和文件夹，拖进去上传（保持文件夹结构）
   - 也可以直接把整个压缩包解压后，用 GitHub Desktop 或网页上传功能整体拖入
6. 点击 **Commit changes** 完成

> 如果你更熟悉命令行，也可以用 `git init && git add . && git commit -m "init" && git remote add origin <你的仓库地址> && git push` 完成，效果一样。

---

## 第二步：创建 Supabase 数据库

1. 打开 [supabase.com](https://supabase.com)，用 GitHub 账号登录最快
2. **New project**：
   - 项目名随便填
   - 数据库密码随便设一个（记下来，但后面用不太到）
   - 区域选离你近的（比如 Singapore / Tokyo）
   - 点 **Create new project**，等 1-2 分钟初始化
3. 进入项目后，左侧菜单点 **SQL Editor**
4. 点 **New query**，把项目里 `supabase_schema.sql` 文件的全部内容复制粘贴进去
5. 点右下角 **Run**，看到 "Success" 就说明建表成功了
6. 左侧菜单点 **Project Settings**（齿轮图标）→ **API**
7. 这里有两个值待会儿要用：
   - **Project URL**（形如 `https://xxxx.supabase.co`）
   - **anon public** key（一长串字符）
   - 先复制到记事本备用

---

## 第三步：获取 Anthropic API Key

1. 打开 [console.anthropic.com](https://console.anthropic.com)，登录
2. 左侧 **API Keys** → **Create Key**
3. 复制生成的 key（`sk-ant-` 开头），同样先存到记事本
4. 需要先在账户里充值一点余额（几美元即可，AI 寻路/提纯功能按用量扣费，用量很小）

---

## 第四步：部署到 Vercel

1. 打开 [vercel.com](https://vercel.com)，用 GitHub 账号登录
2. **Add New** → **Project**
3. 找到你刚才创建的 `worldtree` 仓库 → **Import**
4. 在 **Environment Variables** 这一栏，添加三个变量（名字必须完全一致）：

   | 变量名 | 值 |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | 第二步复制的 Project URL |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 第二步复制的 anon public key |
   | `ANTHROPIC_API_KEY` | 第三步复制的 API key |

5. 点 **Deploy**，等 1-2 分钟
6. 部署完成后会给你一个网址，形如 `https://worldtree-xxx.vercel.app`，打开就是你的网站，手机电脑都能访问

---

## 之后想改代码怎么办？

以后每次你在 GitHub 仓库里改动文件（比如编辑 `app/page.js`），Vercel 会自动重新部署，几十秒后网站自动更新，不需要重新走一遍上面流程。

## 想绑定自己的域名？

Vercel 项目页面 → **Settings** → **Domains**，按提示添加你自己的域名并配置 DNS 即可，免费。

## 遇到问题？

- 打开浏览器开发者工具（F12）看 Console 报错信息
- 检查 Vercel 项目的 **Deployments** 里构建日志是否报错
- 常见问题多是环境变量名打错了一个字符，仔细核对第四步的三个变量名
